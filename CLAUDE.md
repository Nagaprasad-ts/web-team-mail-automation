# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# First-time setup
composer setup

# Development (runs Laravel server, queue listener, and Vite HMR concurrently)
composer dev

# Run tests (includes Pint lint check, PHPStan, and Pest)
composer test

# Run a single test or filter
php artisan test --compact --filter=TeamTest

# PHP formatting (auto-fix)
composer lint                   # or: vendor/bin/pint --dirty --format agent

# TypeScript type check
npm run types:check

# Frontend lint + format
npm run lint && npm run format

# Full CI check (lint, format, types, tests — no auto-fix)
composer ci:check
```

## Architecture

**Stack:** Laravel 13 · Inertia v3 · React 19 · TailwindCSS 4 · Fortify · Wayfinder · SQLite (dev)

### Backend

- **Authentication** is handled entirely by Fortify. Actions live in `app/Actions/Fortify/`. The `User` model includes passkey support and 2FA columns.
- **Team multi-tenancy:** Users belong to many teams via a `memberships` pivot. `HasTeams` concern (`app/Concerns/`) is mixed into `User`. The currently active team is tracked via `users.current_team_id`.
- **Roles/permissions** are PHP-backed enums: `app/Enums/TeamRole.php` and `app/Enums/TeamPermission.php`. Authorization is enforced via policies in `app/Policies/` and the `EnsureTeamMembership` middleware.
- **Routes** are split across `routes/web.php` (public + dashboard + teams) and `routes/settings.php` (included from web.php).
- **Data objects** in `app/Data/` (e.g., `UserTeam`, `TeamPermissions`) are used for typed Inertia prop passing.

### Frontend

- **Pages** live in `resources/js/pages/`. Inertia renders them server-side routed — no React Router.
- **Layouts** are set via `setLayoutProps` / `useLayoutProps` (Inertia v3 pattern). Main authenticated layout: `resources/js/layouts/app-layout.tsx`. Auth pages use `auth-layout.tsx`.
- **Route helpers** are auto-generated by Wayfinder into `resources/js/routes/` and `resources/js/actions/`. Regenerate after adding routes: `php artisan wayfinder:generate`.
- **UI components** are in `resources/js/components/ui/` (Radix UI primitives wrapped with TailwindCSS). Check here before building new components.
- **Toast notifications** use Sonner (`resources/js/components/ui/sonner.tsx`).

### Key Conventions

- Team invitations are created with an expiry and accepted/declined via signed URLs.
- Soft deletes are enabled on `Team`. Restore is not currently exposed in the UI.
- `app/Concerns/GeneratesSlug.php` handles unique slug generation for teams.
